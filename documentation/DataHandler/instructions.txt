Сценарии взаимодействия:

1. Частичное или полное заполнение jsonstore с заменой существующих (обновление) или несуществующих (запись) данных

внешний класс --(перечень данных для изменения/записи)--> **DataHandler**
DataHandler --(запрос данных)--> **DataGetter**
DataGetter --(полученные данные + execution code: 2200‑2205)--> DataHandler
--(через DataHandler)--> **DataParser**
DataParser --(обработанные данные + execution code: 2300‑2304)--> DataHandler
--(через DataHandler)--> **Saver**
Saver --(обработанные данные)--> jsonstore
Saver --(execution code: 2400‑2404)--> DataHandler
DataHandler --(execution code: 2100‑2108)--> внешний класс

2. Запрос данных из jsonstore

внешний класс --(тег клана)--> **DataHandler**
DataHandler --(запрос данных)--> **Loader**
Loader --(данные из jsonstore + execution code: 2500‑2503)--> DataHandler
DataHandler --(данные + execution code: 2100, 2106 или соответствующий)--> внешний класс

3. Автоматические события: проверка свежести данных и обновление

внешний класс --(запрос на проверку свежести данных)--> **DataHandler**
DataHandler --(запрос проверяемых данных)--> **Loader**
Loader --(данные + execution code: 2500‑2503)--> DataHandler
--(через DataHandler)--> **UpdateDecider**
UpdateDecider --(решение + execution code: 2600‑2602)--> DataHandler
   если **2601** (данные устарели) → выполняем п.1 и отправляем execution code (2100‑2108) во внешний класс
   если **2600** (данные актуальны) → DataHandler --(execution code: 2101)--> внешний класс
   если **2602** (ошибка определения свежести) → DataHandler --(execution code: 2107)--> внешний класс
