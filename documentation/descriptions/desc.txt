Назначение проекта
Разработка приложения для отслеживания активности участников клана в игре Clash of Clans. Цель — автоматизировать сбор метрик, расчёт рейтинга активности и предоставить главе клана инструменты для принятия решений (поощрение активных, исключение неактивных).

Периоды учёта активности
- Длительность периода — один месяц.
- Начало периода — 11 число месяца.
- Конец периода — 10 число следующего месяца.
- Рейтинг рассчитывается по окончании периода, но возможны досрочные исключения при наступлении определённых условий.

**Структуры данных, используемые в GUI**

Данные для отображения подготавливаются в трёх основных форматах:

1. **MembersTableData** — список участников клана для левой таблицы.
   Каждый элемент — словарь с ключами:
   - `tag` — тег игрока (строка)
   - `name` — никнейм (строка)
   - `activity_points` — рейтинг активности от 0 до 100 (целое)

2. **PlayerFrameData** — подробная информация об игроке для центральной карточки.
   Каждый элемент — словарь, содержащий:
   - Базовые поля: `tag`, `name`, `level`, `th_level`
   - Активность за период: `cg_points` (очки ИК), `CCcontributions` (формат `"пожертвовано/получено"`), `buildings_upgraded` (количество улучшений), `days_offline`, `capital_contributions`
   - Сводки по событиям:
     - `cwl`: словарь с ключами `participation` (bool), `attacks` (int), `stars` (int)
     - `cw`: словарь с ключами `participation` (общее число участий), `with_attacks` (число войн с атаками), `no_attacks` (число войн без атак)
     - `raids`: словарь с ключами `participation` (число рейдов с участием), `attacks` (сумма атак), `available_attacks` (сумма доступных атак)

3. **ExtraPanelData** — детальные данные для правой панели.
   Каждый элемент — словарь с ключами:
   - `tag` — тег игрока
   - `raids`: список рейдов с полями `id`, `date`, `attacks`, `available_attacks`, `gold_earned`
   - `cw`: список клановых войн с полями `id`, `date`, `result` (`win`/`lose`/`tie`), `participation`, `attacks`, `stars`
   - `cwl`: словарь с данными о ЛВК: `id`, `date`, `days` (список дней с полями `day`, `replaced`, `attack`, `stars`, `percentage`)

---

**Модули получения и хранения данных**

- **DataGetter** (cocapi.py) — асинхронный клиент для официального API Clash of Clans. Реализует методы:
  - `get_players_data(clantag)` — получает расширенную информацию о текущих участниках клана.
  - `get_raids_data(clantag)` — получает последний рейд-лог (с детализацией по участникам).
  - `get_cw(clantag)` — получает информацию о текущей или завершённой клановой войне.
  - `get_cwl(clantag)` — получает данные о текущей Лиге Войн Кланов (группа, войны, атаки).

- **StorageCreator** (storage_creator.py) — создаёт файловую структуру для хранения исторических данных. Для каждого клана создаются папки:
  - `players/current_members` — данные текущих участников по снимкам.
  - `players/left_players` — данные покинувших игроков.
  - `cw`, `cwl`, `raids`, `clan_games` — подпапки с соответствующими логами и timestamp-файлами.

**Интерфейс приложения (GUI)**

Главное окно разделено на три панели:

1. **Левая панель — MembersTable** (members_table.py)
   Таблица участников клана с колонками:
   - № (порядковый номер, игнорирует сортировку)
   - Тег игрока
   - Никнейм
   - Актив (рейтинг активности от 0 до 100) — сортируемый столбец
   При выборе строки испускается сигнал с тегом выбранного игрока.

2. **Центральная панель — PlayerFrame** (player_frame.py)
   При выборе игрока в таблице отображается карточка с подробностями:
   - **Header**: Тег, ник, уровень, уровень ратуши.
   - **Activity_labels** (с кнопкой редактирования для очков ИК):
     * Очки игр кланов
     * Пожертвования/получено войск
     * Количество улучшений
     * Дней офлайн
     * Пожертвовано золота столицы
   - **CWL_group**: участие (Да/Нет), количество атак, количество звёзд + кнопка «Подробнее» (открывает детальную информацию о ЛВК в ExtraPanel).
   - **CW_group**: количество войн с участием, без атак, с атаками + кнопка «Подробнее».
   - **Raids_group**: количество рейдов с участием, общее число атак, доступные атаки + кнопка «Подробнее».
   Кнопки «Подробнее» меняют текст на «→» при активации и блокируются до выбора другой секции.

3. **Правая панель — ExtraPanel** (extra_info_panel.py)
   Изначально пустая. При выборе игрока и нажатии одной из кнопок «Подробнее» отображает детальную информацию:
   - **ЛВК**: таблица с колонками «День» и «Информация» (цветовая индикация: зелёный — 3 звезды, жёлтый — 1–2 звезды, светло-жёлтый — 0 звёзд, серый — заменён, красный — пропуск).
   - **КВ**: таблица с колонками «id», «Дата», «Атаки», «Звёзды». Строки окрашиваются в зависимости от результата войны (зелёный — победа, красный — поражение, жёлтый — ничья, серый — неучастие).
   - **Рейды**: таблица с колонками «id», «Дата», «Атаки», «Золото». Строки окрашиваются по числу атак (зелёный — все атаки, жёлтый — частично, красный — 0).

4. **StartupPanel** (startup_panel.py) — начальный экран с полем ввода тега клана и кнопкой «Получить информацию». При успешном вводе заменяется основным интерфейсом.

**Правила оценки активности и исключения** (из vision-файлов, пока не реализованы в коде)

*Условия для досрочного исключения (до окончания периода):*
- Результат ИК больше 0, но меньше 200 очков.
- Пропуск 3 КВ подряд будучи участником (счётчик сбрасывается, если перед войной игрок поставил красный знак или предупредил в чате).
- Непрерывный офлайн более 7 дней при условии, что клан заполнен (45–50 участников).
- Пропуск 3 рейдов подряд.

*Условия для исключения по окончании периода:*
- Пропуск всех рейдов за период.
- Результат ИК менее 1000 очков.
- Участие в ЛВК при заявленном согласии, но отсутствие атак.
- Общее количество дней офлайн более 10.
- Нулевые пожертвования войск.
- Ноль участий в КВ (с атаками).
- Отсутствие улучшений зданий.

*Относительные показатели (влияют на рейтинг, но не являются прямым основанием для кика):*
- Количество пропущенных КВ подряд.
- Давность последнего входа.
- Количество пропущенных рейдов подряд.

**Дополнительные правила управления (из видения главы клана)**
- По умолчанию в КВ участвуют все, у кого нет красного знака в профиле. Исключение — отказ в заявке или предупреждение в чате.
- В ЛВК состав формируется по результатам активности перед началом лиги.
- Минимальные ожидания: хотя бы 1 атака в каждой КВ (при участии), 2–3 атаки в рейдах, 1000 очков в играх кланов.
- Повышение до старейшины даёт «щит»: при нарушении активности старейшина сначала разжалуется, а уже потом может быть исключён.
- При разгромной победе в КВ отсутствие атак у некоторых участников может не учитываться; при поражении требования строже.
- В играх клана нулевой результат при неучастии (не брал задания) не вызывает вопросов; низкие результаты (1–150) требуют компенсации высокой активностью в другом.
- Присоединившиеся в конце ИК для получения наград без вклада могут быть исключены.
- Длительный офлайн (более недели) ведёт к исключению только при заполненном клане или большом количестве неактивных.

**Версии приложения**
1. Версия 1 — отслеживание активности только в текущем периоде (реализована в коде, данные генерируются для одного периода).
2. Версия 2 — добавление истории активности по периодам (планируется).
3. Версия 3 — анализ эффективности атак и результатов (планируется).
